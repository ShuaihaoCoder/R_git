# Code Map

生成帮助用户理解代码实际运行方式的中文 Markdown 路线图。根据代码规模决定图的数量和细节，不要把简单代码强行画成复杂项目。

## 工作流程

1. 阅读代码入口、主要文件和现有文档，确认真实运行路径。
2. 区分实际运行文件、工具文件、参考文件和历史版本。
3. 找出真正发生的调用、输入、返回值、共享变量和最终结果。
4. 根据项目复杂度选择最小但足够清楚的图组。
5. 使用 Mermaid 绘图，并在图外补充必要的中文解释或变量表。
6. 检查图与当前代码一致，且 Mermaid 代码块完整。

## 先判断复杂度

绘图前先判断代码属于哪一级，优先选择较简单的级别。

### 简单代码

适用于单文件脚本、少量函数或单向运行流程。

- 通常只画 1 张主路线图。
- 展示入口、关键函数、输入、输出和最终结果。
- 只有变量变化特别重要时，才增加 1 个简短变量表或例子。
- 不为每个函数单独画图。

### 中型项目

适用于多个文件、存在公共 helper、数据加载或少量用户交互的项目。

- 画 1 张项目总图。
- 为 2 至 5 个关键文件或关键流程分别画图。
- 增加 1 个初始运行例子，以及 1 个典型后续输入例子。
- 只展开真正影响理解的函数和变量。

### 复杂项目

适用于多入口、多层文件调用、事件驱动、reactive、缓存、后台任务或大量方法的项目。

- 画项目总图、关键变量传递表和主要文件路线图。
- 展示初始运行、后续输入、重新运行或错误处理。
- 需要时为每个重要运行文件建立单独章节。
- 读取 [references/complex-project-code-guide.md](references/complex-project-code-guide.md)，将其作为完整项目的风格参考，而不是必须照搬的模板。

## 图中必须表达的内容

路线图重点展示代码后来真正运行时发生的事情，而不只是罗列定义。

- 哪个文件或用户操作启动流程；
- 后来在哪里调用函数；
- 调用时传入什么；
- 函数返回什么；
- 返回值交给谁；
- 哪个重要变量发生变化；
- 变量变化后，哪个页面、模型、文件或结果跟着变化。

箭头文字使用明确内容，例如：

- `example_id + data_bundle`
- `完整 case list`
- `input$method_link_var`
- `selected_method: linear_regression -> var`

避免只写“调用”“处理”“返回”等无法说明数据内容的箭头。

## 单文件路线图

只有在文件对整体运行有独立作用时，才为它单独画图。说明：

- 文件在项目中的作用；
- 由谁加载或运行；
- 主要功能后来在哪里使用；
- 输入来自哪里；
- 返回值或修改的变量影响什么结果。

不需要专门画“函数在这里被定义”。重点画它后来被调用时的输入、输出和影响。

对于不参与实际运行的文件，明确标记“仅供参考”或“历史版本”，不要让读者误以为它会改变当前结果。

## 运行案例

根据项目实际情况选择案例，不要求每次全部绘制。

### 初始运行

展示从入口到最终结果的顺序，例如：

`启动 -> 加载依赖 -> 读取数据 -> 运行分析 -> 建立结果 -> 打开页面`

复杂项目可同时列出关键变量的初始状态。

### 后续输入

使用一个具体操作展示因果关系，例如：

`点击 VAR -> input 改变 -> open_method("var") -> selected_method 改变 -> 打开 Method Detail -> 显示 VAR 结果`

变量变化重要时，写出变化前后的真实值。

### 重新运行、缓存或错误处理

只有项目存在这些机制时才绘制。展示缓存如何读取或清除、模型如何重新计算、fallback 如何改变结果。

## Mermaid 风格

默认使用较大的图，确保在 Markdown、GitHub 或 Obsidian 中容易阅读。

流程图推荐配置：

```mermaid
%%{init: {'flowchart': {'nodeSpacing': 55, 'rankSpacing': 85}, 'themeVariables': {'fontSize': '17px'}}}%%
flowchart TB
```

时序图推荐配置：

```mermaid
%%{init: {'sequence': {'actorMargin': 60, 'messageMargin': 45}, 'themeVariables': {'fontSize': '16px'}}}%%
sequenceDiagram
```

- 使用 `flowchart TB` 展示文件关系、函数路线和变量影响。
- 使用 `flowchart LR` 展示较短的输入输出链。
- 使用 `sequenceDiagram` 展示启动顺序、用户点击和跨文件调用。
- 图过大时拆成总图和局部图，不要缩小字体硬塞。
- 简单代码不需要添加复杂 Mermaid 初始化配置以外的额外装饰。

## 文字风格

- 使用中文解释，保留真实函数名、变量名和文件名。
- 使用大白话说明技术行为，并给出具体输入输出。
- 解释实际代码，不写抽象教材式定义。
- 先讲运行路线，再讲细节。
- 不声称代码实现了未经验证的行为。
- 不为了绘图修改原始运行代码。

## 验证

完成后检查：

- 绘图规模是否与代码复杂度匹配；
- 实际运行文件和参考文件是否区分清楚；
- 重要调用是否说明输入、输出和下一步；
- 关键变量是否在必要时展示前后变化；
- Mermaid 代码块是否成对闭合；
- 图中文字是否足够大；
- 文档描述是否与当前代码一致。
